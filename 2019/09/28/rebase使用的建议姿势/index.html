rebase使用的建议姿势

2019-09-28

通过一个场景来说明rebase的最佳使用姿势：

假设有一个master分支，现在来了一个新需求，增加一个支付功能，因此需要切一个feature/pay分支，然后开发完成之后需要将feature/pay分支合并到master分支（一般不会直接合并到master分支的，这里简化处理），操作步骤如下：

  1. 切换到master分支，pull最新代码；
  2. 切换到feature/pay分支，执行git rebase -i HEAD~5,合并5个commits；
  3. 执行git rebase master，将master最新代码同步到feature/pay分支。可能会有冲突，手动解决完冲突后 ，执行git rebase --continue；
  4. 切换到master分支，执行git merge feature/pay，将分支代码合并到master；
  5. push master到远程仓库。

rebase和merge的使用建议：

一般主分支合并到其他分支使用rebase，其他分支合并到主分支使用merge。
